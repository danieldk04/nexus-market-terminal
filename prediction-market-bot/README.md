## 🎯 Signal Generation Strategy

Signals are generated by combining:

1. **Twitter Sentiment** (30% weight)
   - Volume of mentions
   - Sentiment polarity
   - Influencer activity

2. **Reddit Sentiment** (20% weight)
   - Post engagement
   - Upvote ratios
   - Comment volume

3. **News Sentiment** (25% weight)
   - Source credibility
   - Recency weighting
   - Headline sentiment

4. **Market Liquidity** (15% weight)
   - Depth analysis
   - Volume trends
   - Bid-ask spread

5. **Historical Accuracy** (10% weight)
   - Past signal performance
   - Category-specific accuracy

**Minimum requirements for signal:**
- 3+ data sources with confidence >0.3
- Overall confidence ≥70%
- Liquidity ≥$100K

## 🛡 Risk Management

### Position Sizing (Kelly Criterion)
```python
Kelly % = (Win% × Avg Win - Loss% × Avg Loss) / Avg Win
Position Size = Portfolio Value × Kelly% × 0.25  # Fractional Kelly
```

### Risk Controls
- **Stop Loss**: -15% per position
- **Daily Loss Limit**: -20% portfolio
- **Max Exposure**: 50% of capital
- **Max Positions**: 10 concurrent
- **Correlation Limit**: Max 3 in same category

## 🚨 Troubleshooting

### Issue: Low win rate (<60%)
**Solution:**
1. Increase `min_confidence` to 0.75+
2. Add more data sources
3. Review signal weights in config

### Issue: API rate limits
**Solution:**
1. Increase scrape intervals
2. Implement API key rotation
3. Enable caching

### Issue: High slippage
**Solution:**
1. Reduce position sizes
2. Use limit orders
3. Enable order splitting
